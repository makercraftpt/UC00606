Anatomia de uma função

Cada parte tem um nome e um papel

def verificar_login(utilizador, password):
    """Verifica credenciais. Devolve True se válido."""
    if utilizador == "admin":
        return True
    return False
palavra-chave def
nome snake_case, diz o que faz
parâmetros o que a função recebe
docstring descreve o propósito
return o que a função devolve
resultado = verificar_login("admin", "cyber2025")   # ← chamar a função

Porquê usar funções?

Três princípios fundamentais da programação estruturada

Não repetir (DRY)
Escreve o código uma vez, usa-o quantas vezes precisares.
Organizar e ler
Divide um problema complexo em partes simples.
Reutilizar
A mesma função serve em programas diferentes.

Parâmetros — o que as funções recebem

Obrigatórios, opcionais e com valor por defeito

# Obrigatório
def saudar(nome):
    print(f'Ola, {nome}!')

# Com valor por defeito
def saudar(nome, nivel='RECRUTA'):
    print(f'Ola, {nome} [{nivel}]!')

saudar('Ana')
saudar('Ana', 'ADMIN')
Obrigatório
Tem de ser sempre passado. Erro se faltar.
Por defeito
Se não for passado, usa o valor indicado.
Ordem
Parâmetros sem defeito vêm sempre primeiro.

return — o que as funções devolvem

Um valor, nenhum, ou vários ao mesmo tempo

def quadrado(n):
    return n * n

r = quadrado(5)
print(r)  # 25
def minmax(lst):
    return min(lst), max(lst)

a, b = minmax([3,1,4])
# a=1  b=4
def mostrar(msg):
    print(msg)
    # sem return = None

r = mostrar('Ola')
print(r)  # None

Todas as funções sem return devolvem None automaticamente.

return como sinal de estado

Usar o valor de retorno para indicar sucesso ou falha

def fazer_login(user, pw):
    if user == 'admin' and pw == 'cyber2025':
        return True   # sucesso
    return False       # falha

if fazer_login('admin', 'cyber2025'):
    print('Acesso concedido')
else:
    print('Acesso negado')
  • return 0 → sucesso (estilo C) · return -1 → erro genérico
  • return None → sem resultado · return msg → mensagem de erro

Âmbito — onde vivem as variáveis

Variáveis criadas dentro de uma função só existem dentro dela

sistema = 'CyberBase'   # global — existe em todo o programa

def iniciar():
    mensagem = 'Sistema iniciado'  # local — só existe aqui
    print(sistema)    # OK — acede ao global
    print(mensagem)   # OK — local existe aqui

iniciar()
print(sistema)    # OK
print(mensagem)   # ERRO — mensagem nao existe aqui!

Boas práticas

Convenções que tornam o código mais legível e profissional

✔ verificar_login() — nome com verbo
✔ fazer_login() — uma função, uma tarefa
✔ ≤ 20 linhas — fácil de ler e testar
✔ docstring sempre, mesmo que só uma linha

Funções noutras linguagens

ConceitoCJavaScriptPython

O que aprendemos hoje

def parametros return ambito docstring snake_case DRY

  • Uma função = uma responsabilidade
  • Nome com verbo em snake_case — verificar_login(), calcular_media()
  • Documenta sempre com docstring e cabeçalho de ficheiro
  • O return pode indicar sucesso, falha ou um resultado calculado
  • Variáveis locais não existem fora da função — âmbito importa
