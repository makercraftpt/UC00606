Desafio: porque é que isto falha?

Observa o código antes de vermos a teoria

log = 'IP=192.168.1.45 ERRO'
partes = log.split(' ')
ip = partes[0][3:]
print(ip)

# ... noutro dia, log muda ...

log = 'ERRO IP=192.168.1.45'
partes = log.split(' ')
ip = partes[0][3:]
print(ip)   # ERRADO!
  • 1. partes[0][3:] assume que a ordem é sempre a mesma
  • 2. Se a ordem das palavras mudar, o código parte
  • 3. Como encontrar o IP em qualquer posição? → regex

Anatomia de um regex

Cada símbolo tem um papel próprio

match = re.search(padrao, texto)

if match:
    print(match.group())
else:
    print('sem padrao encontrado')
variável guarda o resultado (Match ou None)
re.search() procura o padrão no texto
padrão regex o que estamos à procura
verificação confirma se encontrou algo

Porquê usar regex?

Três motivos para dominar padrões em texto

Poupar tempo
Um padrão, muitos textos. Escreves o padrão uma vez, usa em qualquer log ou ficheiro.
Validar dados
Confirma se um email, IP ou password tem o formato certo — sem dezenas de if.
Extrair dados
Tira IPs, datas e emails de logs inteiros em segundos.

Slicing — como cortar uma string

Índices, intervalos e o passo (step)

# texto[inicio:fim:passo]
ip = '192.168.1.45'
print(ip[0:3])     # '192'

# indices negativos contam do fim
print(ip[-2:])     # '45'
print(ip[::-1])    # invertido

print(ip[:3])
print(ip[4:])

💡 Slicing nunca dá erro fora dos limites — devolve uma string vazia ou parcial.

re.search() como sinal

Usar o resultado para saber se encontrou ou não

texto = 'login OK, IP 10.0.0.5'
m = re.search(r'\d+\.\d+\.\d+\.\d+', texto)

if m:            # encontrou = Match
    print('IP encontrado')
else:            # None = não encontrou
    print('sem IP no texto')
  • m.group() → o texto encontrado
  • m.start() → posição inicial
  • m.span() → (início, fim)

search() vs findall() — quanto procuram

search() para no primeiro match; findall() percorre tudo

logs = 'IP 1.1.1.1 e IP 2.2.2.2 falharam'
padrao = r'\d+\.\d+\.\d+\.\d+'

m = re.search(padrao, logs)       # só o 1º
print(m.group())                  # '1.1.1.1'

todos = re.findall(padrao, logs)  # todos
print(todos)
# ['1.1.1.1', '2.2.2.2']

Boas práticas

Convenções que tornam os padrões mais legíveis e seguros

✔ r'\d+\.\d+' evita barras duplas
✘ '\\d+\\.\\d+' confuso, erro fácil
✔ padrao = re.compile(r'\d+')
✘ recompilar o padrão dentro de um loop
✔ grupos nomeados: (?P<ip>\d+...)
✘ padrões gigantes numa só linha
✔ testar o padrão em regex101.com
✘ usar sem testar — falha em silêncio

Padrões comuns — cheatsheet regex

Os padrões que vais usar mais nesta aula

IP:        \d{1,3}(\.\d{1,3}){3}
Email:     \S+@\S+\.\S+
Timestamp: \d{2}:\d{2}:\d{2}

\d = dígito       \S = não-espaço
+  = 1 ou mais    {n} = exatamente n
log = '14:32:07 IP=10.0.0.5'
hora = re.search(r'\d{2}:\d{2}:\d{2}', log)
ip = re.search(r'\d{1,3}(\.\d{1,3}){3}', log)

print(hora.group())  # '14:32:07'
print(ip.group())    # '10.0.0.5'

import re
padrao = r'\d{1,3}(\.\d{1,3}){3}'

# funciona sempre!

Missão v0.6 — preview

O simulador agora analisa logs com regex

def analisar_logs(logs):
    """Extrai IPs e horas suspeitas."""
    p_ip = r'\d{1,3}(\.\d{1,3}){3}'
    p_hora = r'\d{2}:\d{2}:\d{2}'
    for linha in logs:
        ip = re.search(p_ip, linha)
        hora = re.search(p_hora, linha)
        if ip:
            print(hora.group(), ip.group())
    return len(logs)

O que aprendemos hoje
